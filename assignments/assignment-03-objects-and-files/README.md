# Assignment 03 — Pet Show & Road Profiler

## Overview

This assignment introduced two important programming ideas: **objects** and **file input**. Pet Show used objects to represent animated animals, while Road Profiler read height data from a file and analysed the road profile.

## What I Built

- Pet Show animation
- Animal movement routines
- Road Profiler
- Height data analysis and display

## My Approach

For Pet Show, I worked with `Animal` objects and created routines that reused movement and speech methods. For Road Profiler, I loaded height values from a text file into a list, then used separate methods to calculate and display information such as average and maximum height.

## Key Takeaways

- Objects are useful when a program needs to represent real things with their own data and behaviour.
- File input allows a program to work with external data instead of only user input.
- Lists make it easier to process a collection of values.

## Looking Back

If I revisited this assignment today, I would:

- Make the animal routines more flexible.
- Use clearer method separation in the data analysis section.
- Add error handling for missing or invalid files.
- Improve comments around the file-processing logic.
